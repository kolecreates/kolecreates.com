## What is fullstack? 

Fullstack is a term for a combination of a backend (e.g. server, database, network) and a frontend (e.g. website, mobile app). Fullstack developers are those who work on both.

## The modern fullstack

The modern fullstack is a stack built using the latest app development frameworks and cloud services. 
The term [Jamstack](https://jamstack.org/) has emerged to describe a stack using the most recent technologies. The Jamstack approach is popular because it bootstraps an application with best practices and powerful functionality.

## What's next?

The suite of fullstack technologies is rapidly evolving. The next generation of websites, mobile apps, and web services is being built upon hyper-abstract frameworks and super-powered cloud services. It's natural to assume the trend will continue and within a few years the jamstack will become .... I don't even know.
One thing I forsee is low-code solutions spreading and improving and operating on more of the stack.

## What's the next, next thing?

### The XY Problem

[The XY Problem](https://xyproblem.info/) refers to asking a question about your attempted solution (Y) rather than the problem you are trying to solve (X). Any experience developer is intimately familiar with this. Interestingly, the XY problem can be seen outside question-answer forums. In fact, many products, services, and open-source projects are solving the Y problem but not the X problem. 
Some examples:
- [Web Bundlers](https://vitejs.dev/guide/why.html) and all their complexity exist because Javascript had no native way to modularize large code bases.
- All-in-one platforms like Salesforce and AWS introduce features that solve a problem specific to the platform or that would otherwise be straightforward to solve.
- [accessiBe](accessibe.com) is a startup doing fantastic work to improve the accessibility of the web. Unfortunately, doing this is a lot harder than it needs to be because web technology and the way it is design is fundamentally flawed in terms of accessibility.
- ...more to come

I hold no judgement for the smart and driven people behind such solutions - because often the solution to X is not clear and/or a solution to Y is more practical.

### Solving the X problem

The next, next thing is a fullstack that solves the X problem. The future fullstack is not even a stack<sup>1</sup> - it's a personalized and dynamic interface to data and computation. 

In formal gibberish, some-sort of turing-complete information processing design system embedded within a network of sibling systems and input/output devices.

Someone uses this interface to describe the problem they want to solve and be guided through the data and computational requirements in order to create the solution once the giant red BUILD-IT button is pressed.
